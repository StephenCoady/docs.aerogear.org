Preparing Push For Use With iOS

Before using Push with iOS, first you need to submit a request for a new digital certificate, which is based on a public/private key. The Certificate itself is acting as the public key, and when you request it, a private key is added to your KeyChain tool. The Cerficate will be used later on, to request an SSL certificate for the Apple Push Network Service, which will allow the AeroGear Push Server to send notification messages to it.

Prerequisites

  • An Apple developer account

  • A Mac

Procedure

  1. Open the KeyChain Access tool found in Applications > Utilities folder on your Mac.

  2. Choose the KeyChain Access menu.

  3. Select Certificate Assistant > Request a Certificate from a Certificate Authority:

  4. Store the CSR on file as it is required later in the provisioning portal when requesting the actual SSL cert.

    Going back to KeyChain Access, there is now a new private key

  5. Open the Provisioning Portal and log in with your Apple developer account.

  6. Click the Identifiers link in order to create a new App ID (use the PLUS Icon on the right).

    If your app is already registered please make sure that push capabilities are added. Now open the App ID and skip to step 13.

  7. In the formular, give the App ID a descriptive name, for example, My first AeroGear Push App.

  8. Ensure that the Push Notifications checkbox is selected:

  9. Enter YOUR bundle ID in the Bundle ID field. In this example we choose org.aerogear.PushTest, however you must use your own ID.

    The Bundle ID has to match the one from the actual iOS application that you are building.

  10. On the next screen, confirm your new App ID, and ensure that the Push Notifications option is enabled.

  11. Click the Submit button to complete this process.

  12. On the next screen, click on the newly created App ID.

  13. The following screen appears:

  14. Click the Edit button and scroll down to the Push Notifications section.

  15. Click the Create Certificate…​ button on the Development SSL Certificate section to generate a Development Certificate:

  16. Perform an upload of the CSR and click the Generate button.

  17. Once complete, click the Edit button and scroll down to the Push Notifications section.

    Notice that the Development SSL Certificate section is filled with the required details:

  18. Create the Production certificate by following the same steps however this time click the 'Create Certificate' button on the Production SSL Certificate section.

  19. On the overview of the App ID, click the Edit button and scroll down to the Push Notifications section.

  20. Now that both Certificates are generated, download both to your computer.

  21. Double click the aps_development.cer file (this is the Development SSL Certificate) and it displays in the KeyChain Access tool:

  22. Export the certificate/private key pair to the .p12 (Personal Information Exchange). Assign a passphrase for it.

    Ensure to store the passphrase in a secure location.
    This file will be uploaded later on to the AeroGear Push Server enabling it to authorize itself for your development application on Apple Push Network Service and send messages to it.

  23. Use the steps above to download the Production SSL Certificate.

  24. Double click the aps_production.cer file (this is the Production SSL Certificate) and it displays in the KeyChain Access tool.

  25. Export the .p12 (Personal Information Exchange).

    Ensure to store the passphrase in a secure location.

  26. Next, you need to create an Apple Provisioning Profile.
