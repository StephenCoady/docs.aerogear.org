Preparing your Application to use

  • You have provisioned one of the Mobile Services as described in provisioning.adoc.

  • You have associated the service with a mobile app record as described in Binding.

Getting the Mobile Services configuration file

  1. Navigate to your project in OpenShift.

  2. On the Overview screen, expand your mobile app record to view the CONFIG INFO.

  3. Copy the configuration to your clipboard.

  4. Save the contents of the clipboard to a new file called mobile-services.json.

Android

Locate this file at the following location in your application project:

src/main/assets/mobile-services.json

iOS

Locate this file at the following location in your application project:

Resources/mobile-services.json

Cordova

Locate this file in your application project and import it using require.

const aerogearConfig = require("./mobile-services.json");

Alternatively you can create a JavaScript object and use directly:

const aerogearConfig = /* Paste here the contents of the clipboard */;
Xamarin

Locate this file at the following location in your application project:

Resources/mobile-services.json

Adding AeroGear the SDK module

Android

SDK is initialized on startup, no further steps required.

iOS

SDK is initialized on startup, no further steps required.

Cordova

  1. Open a terminal and go to your application project root folder.

  2. Run the following command:

    npm install @aerogear/app

  3. In your application’s entry point, add the following code:

    import { init } from "@aerogear/app";

init(aerogearConfig);
Xamarin

SDK is initialized on startup, no further steps required.
